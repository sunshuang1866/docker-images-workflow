## 修复的问题
无需代码修改 — CI 失败类型为 infra-error，根因是 CI 流水线 `eulerpublisher/update/container/app/update.py` 的 appstore 路径预检未对纯文档类 PR 做豁免处理，与代码仓库内容无关。

无

## 修复逻辑
PR #2790 仅修改了仓库根目录的 `README.md` 和 `README.en.md` 两个文档文件，内容为更新基础镜像 Tags 列表。这两个文件本身没有语法或内容错误。CI 失败原因在于 appstore 发布规范预检要求所有变更文件必须位于规范的镜像目录路径下（如 `{场景}/{镜像名}/{版本}/{系统版本}/`），而 README 文件位于仓库根目录，不在任何镜像目录内，因此被误判为 `[Path Error]`。此问题需在 CI 流水线层面修复（如添加白名单机制或区分文档变更与镜像变更），不在本仓库代码范围内。
