# 修复摘要

## 修复的问题
无需代码修改。CI 失败属于 infra-error（基础设施错误），由 openEuler 24.03-LTS-SP4 软件包仓库镜像站的瞬时 HTTP/2 流中断（Curl error 92）导致 `dnf install` 下载 `gcc-c++` 失败。

## 修改的文件
无（infra-error，无需代码修改）

## 修复逻辑
分析报告确认该失败与 PR 变更完全无关。Dockerfile 中的 `dnf install` 命令语法和包名均正确无误（258 个包被列入事务清单，大部分已成功下载）。失败根因是仓库镜像服务器 HTTP/2 服务端不稳定，属于瞬时网络故障。修复方式为重新触发 CI 构建，在镜像恢复稳定后构建即可通过。
