# 修复摘要

## 修复的问题
无需代码修改。CI 失败为基础设施问题（infra-error）：pip 从 `mirrors.aliyun.com` 下载大文件 `nvidia_cudnn_cu13-9.20.0.48`（366 MB）时发生读取超时，已传输至 96.5% 时连接中断，属于远端镜像站临时网络波动。

## 修改的文件
无

## 修复逻辑
分析报告明确指出失败类型为 `infra-error`，根因为阿里云镜像站下载超时，非代码逻辑错误。按照规范，对于 infra-error 类型不进行代码修改。建议重试 CI 构建——该超时发生于大文件下载接近完成时，极有可能为临时网络波动，重试大概率可通过。若多次重试均在同一包超时，再考虑增加 pip `--default-timeout` 参数或更换镜像源。
